Customer Support Ticket Triage System

Overview

This project is a machine learning-based ticket classification system designed to automatically route incoming customer support tickets to the correct department.

It uses classical NLP techniques (TF-IDF + Logistic Regression) instead of large language models to provide:

  • Fast inference
  • Low operational cost
  • High interpretability
  • Deterministic behavior

Problem Statement

Customer support teams receive thousands of unstructured tickets daily. Manually routing these tickets is:

  • Time consuming
  • Error prone
  • Expensive at scale

Incorrect routing increases response time and reduces customer satisfaction.

Solution Architecture

Raw Ticket Text ↓ TF-IDF Vectorization ↓ Logistic Regression Classifier ↓ Predicted Department + Confidence Score ↓ Confidence-Based Routing (Auto / Human Review)

Confidence-Based Routing

If model confidence ≥ threshold → Ticket is auto-routed If model confidence < threshold → Sent for human review

This prevents high-risk misclassification and mirrors real-world production ML deployment practices.

Real-World Applications

  • Banking support ticket routing
  • E-commerce complaint categorization
  • SaaS customer support automation
  • IT helpdesk triage
  • Enterprise internal support systems

This system can reduce operational costs, improve response time, and increase customer satisfaction.

Technology Stack

  • Python
  • Scikit-learn (TF-IDF + Logistic Regression)
  • FastAPI (REST API)
  • HTML + CSS frontend

How To Run

1. Train the model:

python training/train.py

2. Start the API:

uvicorn app.main:app --reload

3. Open the frontend demo page.
